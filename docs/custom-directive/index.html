Custom Directive

Custom directives are classes that manipulates an element in our munster application. Directive is sometimes called as namespace in this framework since the syntax to write directives in view is <namespace>:<namespaceName>={<value>}. The value is not required.

Creating a custom directive#

To create a custom directive. We just need create a class that extends the Directive class provided by the framework.

import { Directive } from '@munster/core';
export class HighlightDirective extends Directive { }

The next step is to add a static property called namespace that has a value of string. This is the namespaces that we want to use in our view.

import { Directive } from '@munster/core';
export class HighlightDirective extends Directive {
public static namespace: string = 'highlight';
}

The next thing we need to do is to loop through the namespaces of an element using the this.eachNamespace method when the directive is initialized.

We will log the value of namespaceName and valueCaller in the browser's console for now to check their values.

import { Directive } from '@munster/core';
export class HighlightDirective extends Directive {
public static namespace: string = 'highlight';
init() {
/**
* namespace syntax <namespace>:<namespaceName>={<value>}
* @param namespaceName - red | yellow
* @param valueCaller - a function that returns the value of the namespace
*/
this.eachNamespace((namespaceName: string, valueCaller: () => any) => {
console.log('namespaceName: ', namespaceName);
console.log('valueCaller: ', valueCaller());
});
}
}

If we use the directive above using this template:

<div highlight:red="sample value"></div>

Then the console should log something like this:

namespaceName: red
valueCaller: sample value

Next, we will implement the methods that we will use to highlight an element. For this example, we will only accept red and yellow namespaceName and for the rest will throw an error.

import { Directive } from '@munster/core';
export class HighlightDirective extends Directive {
public static namespace: string = 'highlight';
init() {
/**
* namespace syntax <namespace>:<namespaceName>={<value>}
* @param namespaceName - red | yellow
* @param valueCaller - a function that returns the value of the namespace
*/
this.eachNamespace((namespaceName: string, valueCaller: () => any) => {
if (this[namespaceName]) {
this[namespaceName](valueCaller);
} else {
directiveNotFoundError(ViewDirectiveClass.namespace, namespaceName);
}
});
}
private red(valueCaller: () => any): void {
this.getElement().style.background = 'red';
}
private yellow(valueCaller: () => any): void {
this.getElement().style.background = 'yellow';
}
}

The namespaces will call its respective class methods once used in the view.

For the example above, highlight:red will call the method this.red with the parameter of valueCaller.

Use the custom directive#

We use the syntax <namespace>:<namespaceName>={<value>} to use a directive.

Using the example directive above we can change the background color of an element by applying a directive to it's opening tag.

<div>
<p highlight:red>I am in a red line.</p>
<p highlight:blue>I am in a blue line.</p>
</div>

Please remember that directive value is optional.

Value change watcher#

To let our directive react when it's a value is changed we can use the this.$watch method of a directive. The watch method has a parameters of value caller, on change callback, and an optional parameter for initial value.

Here's an example on how to change the background color of an element based on the value passed to it namespace.

The directive#

import { Directive } from '@munster/core';
export class HighlightDirective extends Directive {
public static namespace: string = 'highlight';
init() {
this.eachNamespace((namespaceName: string, valueCaller: () => any) => {
if (this[namespaceName]) {
this[namespaceName](valueCaller);
} else {
directiveNotFoundError(ViewDirectiveClass.namespace, namespaceName);
}
});
}
private color(valueCaller: () => any): void {
this.watch(valueCaller, (newValue) => {
this.getElement().style.background = newValue;
}, 'red');
}
}

The component#

<p highlight:color={this.color}>Hello World!</p>

Directive useful methods#

Methods
init()This method is called once the directive is initialized.
getComponent()This method returns the current component where the directive is used.
getElement()This method returns the element where the directive is used.
getNamespaces()This method returns an object that contains the namespaces of an element.
eachNamespace()This method loops through all the value of the namespaces.
watch()This method is used to watch if the value of a namespace is changed.
