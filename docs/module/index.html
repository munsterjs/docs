Module

Module is a way to group components, plugins and other part of application that are related to each other.

Here's an example on how to define the most basic module in munster:

import { Module } from '@munster/core';
@Module({})
export class AppModule { }

Bootstrap component#

Some module has a root component which will be displayed when a module is mounted to the dom tree.

Here's an example on how to bootstrap a component:

import { Module } from '@munster/core';
import { AppComponent } from './app.component';
@Module({
bootstrap: [AppComponent, 'app-root']
})
export class AppModule { }

In this example, the AppComponent is the root component of the module.

Mount to dom tree#

Module must be mounted to the dom tee first before we can interact with its components.

Here's an example on how to mount the module to a dom tree:

import { Module } from '@munster/core';
import { AppComponent } from './app.component';
@Module({
bootstrap: [AppComponent, 'app-root']
})
export class AppModule { }

In the example code above, we bootstrap the AppComponent and set the selector to app-root. The module will now be mounted to the dom tree when we call <app-root /> in the template.

Register a component#

We can register a component to a module in order for this component to use the directives, plugins and other configs of the module.

Here's an example on how to register a component:

import { Module } from '@munster/core';
import { AppComponent } from './app.component';
@Module({
components: [
AppComponent
]
})
export class AppModule { }

To declare a component, we just pass the component to the components array in a module.

Module prefix#

The prefix of a module will be the prefix of the component selector of all the components registered in a module when mounting a component to dom tree.

Here's an example on how to set a prefix to a module:

import { Module } from '@munster/core';
@Module({
prefix: 'custom-prefix'
})
export class AppModule { }

In the example code above, the selector of all the components registered in this module will be <custom-prefix-[component selector] /> when mounting the component to the dom tree. If the prefix is not present, the default prefix will be app.

Submodules#

A module can also have a submodules. Submodules are just another module imported in another module. It is a way to create application with modular architecture and is one way to group and organize your code to make it more reusable and easy to maintain.

Here's an example on how to import a submodule into a module:

import { Module } from '@munster/core';
import { AuthModule } from './auth.module';
@Module({
modules: [
AuthModule
]
})
export class AppModule { }

Components from submodules are automatically available in components registered in the parent module.

Export directives#

Directives are not automatically available to the parents module once a submodule is imported. To make it available in the parent module we need to export the directive.

Here's an example on how to export a directive:

import { Module } from '@munster/core';
import { CustomDirective } from './custom.directive';
@Module({
exports: {
directives: [
CustomDirective
]
}
})
export class AppModule { }
