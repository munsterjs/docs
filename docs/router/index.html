Router

Router is a plugin that enables developers to build a single page application with multiple components that acts as different views of the app. View changes depending on the activated route. Activated routes depends on the url of the browser and the path registered in the router.

How to use#

To use this router, we need to import the RouterModule in order to use the router features inside the parent module.

Here's an example on how to register the router:

import { Module } from '@munster/core';
import { RouterModule } from '@munster/router';
@Module({
modules: [
RouterModule
]
})
export class AppModule { }

Route component#

After the router module is registered in a module, a route component is automatically available to that module. Route component has attributes of path, exact, component, redirect, and property of middleware. Path and component are required and the reset are optional.

Here's an example on how to use the route component:

<app-route path="/sample/path" component="app-sample-component" />

Router properties#

Navigate#

After router is registered in a module, a $navigate property is added to all the component registered in the module. This navigate feature the same as the Navigate plugin.

We can find more information about the navigate plugin in here.

Router#

Component added to the dom tree using the app-route component will have this.$router property. This property is an object that contains some useful property or functions for this router.

The following are the properties and functions inside this.$router property:

Method/Property
onRouteChangeAn observable property that triggers when the routes is changed.
paramsA function that returns the route parameters.
onRouteChange#

Here's an example on how to subscribe to the onRouteChange event:

import Template from './app.component.html';
import { Component, SuperClass, OnConnected, OnDisconnected, SubscriptionInterface } from '@munster/core';
@Component({
selector: 'app-component',
template: Template
})
export class AppComponent SuperClass() implements OnConnected, OnDisconnected {
private subscription: SubscriptionInterface;
public connected(): void {
this.subscription = this.$router.onRouteChange.subscribe(() => {
...
});
}
public disconnected(): void {
this.subscription.unsubscribe();
}
}
params#

We can also get the router parameters using this.$router.params().

Here's an example on how to get the router parameters:

import Template from './app.component.html';
import { Component, SuperClass, OnConnected } from '@munster/core';
@Component({
selector: 'app-component',
template: Template
})
export class AppComponent SuperClass() implements OnConnected {
public connected(): void {
const routerParams = this.$router.params();
}
}

Creating routes#

A route is just a javascript object that contains two required properties, path and component and some optional properties.

Propertiestype
pathStringThe path that should match in the browser url pathname before the route is activated.
componentStringThe component selector of the component that will be rendered where the <app-route /> tag is located.
exactBooleanIf the values is true, then the Component will only activate if route path is an exact match with the browser url pathname but still respect the dynamic route matching.
middlewareArrayIt is another layer of checking if the component should be active or not.

Here are some example on how to create routes:

<div>
<app-route path="/" component="app-landing-page-component" exact={true} />
<app-route path="/login" component="app-login-component" middleware=[GuestMiddleware] />
<app-route path="/auth" component="app-auth-component" middleware=[AuthMiddleware] />
</div>

Dynamic route matching#

Dynamic route matching is a way to match a route path segment into its matching browser url pathname segment. A dynamic segment is denoted by a colon : followed by the segment name. ex. /:userId. The value of the dynamic segments is accessible in it's component only using this.$router.params().

Here's a table of dynamic routes and its corresponding values in this.$router.params():

component pathbrowser url pathnamethis.$router.params()
/:path/100{ path: 100 }
/user/:userId/user/123{ userId: 123 }
/post/:postId/:userId/post/1/123{ postId: 1, userId: 123 }

Router link#

Router link is an additional feature of the router module. It is a directive that allows us to navigate to different routes once an element is clicked.

Here's an example on how to add a router link:

<div>
<a router:link="/route-path">I am a link</a>
<button router:link="/route-path">I am a button</button>
</div>

Middleware#

Middleware is another way to check if a component can activate. It can also run a block of codes before a route event happen.

Here's an example on how to use middleware:

<app-route path="/sample-path" component="app-sample-component" middleware=[SampleMiddleware] />
export class SampleMiddleware {
canActivate() {
// returns a boolean or a promise
}
}

If canActivate method returns true or a promise that resolves to true, then the component is allowed to activate and not allowed if otherwise.
