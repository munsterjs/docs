Component

Components are the most basic building block of an application. Currently, a component is composed of view and logic only but in future releases will include styles.

Structure#

View#

<!-- ./sample.component.html -->
<h1>Sample component view.</h1>

The view of a component is an html file and must a single parent element only but can have multiple child. If needed to have multiple parent, we must wrap it with <fragment> ... </fragment> tag.

Logic#

// ./sample.component.ts
import { Component, SuperClass } from '@munster/core';
import Template from './sample.component.html';
@Component({
selector: 'sample-component',
template: Template
})
export class SampleComponent extends SuperClass() { }

Please take note of the @Component decorator. The template must be imported and passed to the @Component decorator.

Component selector#

Component selector is defined in the @Component decorator.

Here's an example on how to define a component selector:

import { Component, SuperClass } from '@munster/core';
@Component({
selector: 'sample-component-selector',
...
})
export class SampleComponent extends SuperClass() { }

In the example above the tag name of the component will be <[module prefix]-sample-component-selector />.

Define component#

Component must be defined in a module before we can use it. To define a component, we must add it to the components array in its module.

Here's an example on how to define a component in a module:

import { Module } from '@munster/core';
import { SampleComponent } from './sample.component';
@Module({
components: [
SampleComponent
]
})
export class AppModule { }

Custom element define#

Since a component is a web component. It can also be defined using customElements.define.

Here's an example on how to define a web component:

import { SampleComponent } from './sample.component';
customElements.define('sample-component', SampleComponent);
note

Component name must contain a hyphen(-) character or else it will be treated as normal html element.

Data binding#

Data binding is a way to synchronize the data from logic to view and vise versa.

Attribute binding#

Here is an example on how to bind a property from the logic into an attribute of view:

<h1 id={this.attr}></h1>
import Template from './sample.component.html';
import { Component, SuperClass } from '@munster/core';
@Component({
selector: 'sample-component',
template: Template
})
export class SampleComponent {
attr = 'sample-attr';
}

Text binding#

Here is an example on how to display a text from logic to view:

<h1>{greeting}</h1>
import Template from './sample.component.html';
import { Component, SuperClass } from '@munster/core';
@Component({
selector: 'sample-component',
template: Template
})
export class SampleComponent {
greeting = 'Hello World!';
}

Model binding#

view:model

Model binding is a two way binding of data. Every time the model is changed from the view, the value in logic will be updated and the same thing will happen in the view when the model is updated from the logic.

Here's an example on how to bind a model to the view:

<input view:model={this.sampleModel} type="text" />
import Template from './sample.component.html';
import { Component, SuperClass } from '@munster/core';
@Component({
selector: 'sample-component',
template: Template
})
export default class SampleComponent {
sampleModel = '';
}

Hooks#

Hooks are functions that lets you run a block of codes when your component triggers a lifecycle event.

Below are the available lifecycle hooks:

Hooks
connected()Fired when the component is added to the dom.
disconnected()Fired when the component is removed from the dom.
attributeChanged(attrName, oldVal, newVal)Fired when a component attribute is changed.
adopted()Fired when the component is moved from one HTML document to another one with the adoptNode method.
onViewInit()Fired when a component starts building the element of view.
afterViewInit()Fired when a component is done building the element of view.
onChange()Fired when a the change detection in component is triggered.

Child Component#

Child component is a component rendered inside a component. It is rendered by calling its selector as a tag name. Child component can give and receive data from parent component.

Here's an example on how to render a child component:

<div>
<app-child-component></app-child-component>
</div>
note

Child component must be registered to the same module or exported from a child module.

Attributes#

Attributes are values passed down from parent component to child component. Attribute can only hold a value and not objects or arrays.

Here's an example on how to pass a value from parent to child component:

Parent component#
<div>
<child-component
num-attr={this.num}
bool-attr={this.bool}
str-attr={this.str}
></child-component>
</div>
import { Component, SuperClass } from '@munster/core';
import Template './parent.component.html';
@Component({
selector: 'parent-component',
template: Template
})
export class ParentComponent extends SuperClass() {
public num = 123;
public bool = true;
public str = 'hello world';
}
Child component#
import { Component, SuperClass } from '@munster/core';
import Template './child.component.html';
@Component({
selector: 'child-component',
template: Template
})
export class ChildComponent {
@AttrNumber() numAttr;
@AttrBoolean() boolAttr;
@Attr() strAttr;
}
note

The properties in child component and attribute names in template must match using camel casing in component and kebab casing in template.

Watch attribute change#

We can also watch for an attribute change by using the attributeChanged method.

Here's an example on how to watch for an attribute change:

import { Component, SuperClass, OnAttributeChanged } from '@munster/core';
import Template './child.component.html';
@Component({
selector: 'child-component',
template: Template
})
export class ChildComponent implements OnAttributeChanged {
@AttrNumber() numAttr;
@AttrBoolean() boolAttr;
@Attr() strAttr;
public attributeChanged(name: string, oldValue: any, newValue: any): void {
console.log(name, oldValue, newValue);
}
}

Props#

Props are the properties passed down from parent to child. Props is different from attributes since it can hold values, arrays or objects.

Since props is not directly available in web components we need to use a directive for this feature.

Please see the props directive documentation on how to use props.

note

Passing data using attribute is recommended since attribute is using the implementation of web component attributes.
